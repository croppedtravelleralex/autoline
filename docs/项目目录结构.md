# 项目目录结构

本文档提供了 AutoLine 项目的整体目录结构及其各部分功能的简要说明。

## 整体结构概览

```text
autoline/
├── backend/            # 后端 Python (FastAPI) 代码
├── frontend/           # 前端 React (Vite/TS) 代码
├── docs/               # 项目文档（部署教程、目录结构等）
├── scripts/            # 辅助脚本
├── mes_data/           # 模拟数据存储 (SQLite/JSON)
├── README.md           # 项目自述文件
├── render.yaml         # Render 部署配置
└── .agent/             # AI 助手工作流配置
```

---

## 详细目录说明

### 1. Backend (后端)
后端使用 FastAPI 构建，主要负责设备模拟逻辑、状态管理以及历史数据存储。

*   `app/`: 核心代码逻辑
    *   `api.py`: 所有的 HTTP 路由定义（包括状态获取、设备控制、配方管理等）
    *   `models.py`: 系统数据模型定义 (Pydantic models)
    *   `services/`: 核心业务逻辑服务（仿真模拟、状态同步、历史记录等）
*   `main.py`: 后端入口文件，配置 FastAPI 实例及中间件。
*   `Dockerfile`: 容器化配置文件。
*   `zeabur.json`: Zeabur 自动部署配置。
*   `requirements.txt`: Python 依赖清单。

### 2. Frontend (前端)
前端使用 React + TypeScript + Vite + Tailwind CSS 构建。

*   `src/`: 源代码
    *   `components/`: 可复用 UI 组件（仪表盘卡片、控制面板、图表等）
    *   `pages/`: 页面组件（首页、监控页、设置页等）
    *   `services/`: API 请求封装层
    *   `types/`: TypeScript 类型声明
    *   `context/`: 全局状态控制 (Context API)
*   `public/`: 静态资源文件。
*   `index.html`: 单页应用主入口。
*   `zeabur.json`: Zeabur 前端部署配置。
*   `tailwind.config.js`: Tailwind 样式配置。

### 3. Docs (文档)
*   `Zeabur部署教程.md`: 针对 Zeabur 环境的详细部署指南。
*   `Vercel部署教程.md`: 针对 Vercel 环境的部署说明（由于后端限制，目前主要用于前端预览）。
*   `项目目录结构.md`: 即本文件。

### 4. 其他
*   `mes_data/`: 存放模拟生成的历史数据库 `history.db` 和快照。
*   `scripts/`: 包含各种环境下的测试或构建脚本。
*   `参数/` & `MES参数/`: 可能包含一些配置或参考参数。
